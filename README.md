# 📁 FolderTimeEditor — 文件夹/文件时间属性批量修改工具

> **批量修改 Windows 文件夹和文件的创建时间、修改时间、访问时间**
>
> 版本 1.0.0 | Windows 7+ | 绿色单文件版 (无需安装)

![软件截图](docs/screenshot.png)

---

## ✨ 功能特性

- ✅ **批量修改** — 同时修改多个文件夹和文件的时间属性
- ✅ **三种时间戳** — 支持修改 **创建时间**、**修改时间**、**访问时间**
- ✅ **多种设置模式**
  - **统一设置** — 所有选中项设为同一时间
  - **相对偏移** — 基于当前时间 ± 指定偏移量
  - **文件命名提取** — 从文件名中自动提取时间（如 `2024-01-15_报告.docx`）
  - **目录命名提取** — 从父文件夹名称中提取时间
  - **EXIF 读取** — 从照片/图片的 EXIF 信息中提取拍摄时间
- ✅ **递归处理** — 可选包含子文件夹中的文件
- ✅ **拖放支持** — 直接拖拽文件夹/文件到窗口中
- ✅ **操作预览** — 修改前预览将要变更的时间值
- ✅ **撤销保护** — 修改前自动备份，支持撤销操作
- ✅ **进度显示** — 实时进度条，大数量操作不卡死
- ✅ **现代界面** — 基于 `ttkbootstrap` 的美观 Windows 原生风格界面
- ✅ **绿色便携** — 单文件 EXE，无需安装，不写注册表

## 📥 安装与使用

### 方式一：直接下载 EXE（推荐）

1. 从 [Releases](https://github.com/your-username/FolderTimeEditor/releases) 页面下载 `FolderTimeEditor.exe`
2. 双击运行，**无需安装**，即开即用
3. 安全软件可能首次报未知程序，点击「更多信息」→「仍要运行」即可

### 方式二：从源码运行

```bash
# 1. 克隆仓库
git clone https://github.com/your-username/FolderTimeEditor.git
cd FolderTimeEditor

# 2. 安装依赖
pip install -r requirements.txt

# 3. 运行
python main.py
```

### 方式三：自行打包

```bash
# 安装打包依赖
pip install -r requirements.txt

# 执行打包
pyinstaller build.spec

# 输出文件: dist/FolderTimeEditor.exe
```

## 📖 使用说明

### 快速上手（5 步流程）

```
步骤 1 ── 添加项目
        点击「添加文件夹」或「添加文件」按钮，
        或直接拖拽文件/文件夹到列表中。

步骤 2 ── 选择时间模式
        在右侧面板选择时间设置方式：
        • 统一时间：手动输入一个具体时间
        • 相对偏移：当前时间 ± N 天/时/分
        • 从文件名提取：自动解析文件名中的日期

步骤 3 ── 设定目标时间
        根据选择的模式：
        • 统一时间 → 使用日期时间选择器
        • 相对偏移 → 输入偏移量（天/时/分）
        • 文件名提取 → 选择日期格式模板

步骤 4 ── 选择要修改的时间属性
        勾选需要修改的时间类型：
        ☑ 创建时间
        ☑ 修改时间
        ☑ 访问时间

步骤 5 ── 执行修改
        点击「预览」查看变更结果，
        确认无误后点击「执行修改」。
```

## ⚠️ 注意事项

### 管理员权限
- 普通文件修改时间**不需要**管理员权限
- 修改 `C:\Windows`、`C:\Program Files` 等系统目录下的文件，需要**以管理员身份运行**
- 建议：右键 EXE → **以管理员身份运行**，可避免权限不足问题

### NTFS 限制
- **创建时间**只能在 NTFS 文件系统上修改
- FAT32/exFAT（常见于 U 盘、SD 卡）**不支持修改创建时间**
- 程序会自动检测文件系统并给出提示

### 时间精度
- NTFS 时间精度：100 纳秒
- FAT32 时间精度：2 秒
- 程序会自动四舍五入到文件系统支持的最小单位

### 其他
- 修改大量文件（1000+）时请耐心等待进度条完成
- 修改前建议手动备份重要文件
- 程序每次修改时会自动创建撤销信息，重启程序后撤销记录会丢失

## ❓ 常见问题 (FAQ)

### Q: 修改文件夹时间后，里面的文件时间会变吗？
**不会。** 修改文件夹的时间属性**只影响文件夹本身**，不影响其中的文件。如需递归修改，请勾选「包含子文件夹中的文件」。

### Q: 修改时间后能恢复吗？
**可以。** 每次修改前程序会自动记录原始时间。在会话期间，点击「撤销」按钮可以恢复到修改前的状态。但**关闭程序后撤销记录会丢失**。

### Q: 为什么有些文件名的日期提取失败？
文件名日期提取依赖于预定义的日期格式模板。如果您的文件名格式不常见，可以在「格式设置」中添加自定义模板。支持的正则表达式格式包括：
- `YYYY-MM-DD` (如 `2024-01-15`)
- `YYYYMMDD` (如 `20240115`)
- `YYYY_MM_DD` (如 `2024_01_15`)
- `DD-MM-YYYY` (如 `15-01-2024`)

### Q: 程序报错 "Permission denied" 怎么办？
这通常是因为没有足够的文件权限。请尝试：
1. 以管理员身份运行程序
2. 关闭正在使用该文件的程序（如 Word、Excel）
3. 检查文件是否为只读属性

### Q: 可以同时修改文件和文件夹吗？
**可以。** 您可以在列表中添加任意组合的文件和文件夹，程序会统一处理。

### Q: 支持哪些语言？
目前仅支持**中文**界面。后续版本计划添加英文支持。

### Q: 打包时报错 "upx not found" 怎么办？
UPX 是可选的可执行文件压缩工具。如果没有安装 UPX，程序依然可以正常打包，只是生成的 EXE 体积稍大（约 15-20 MB 而不是 8-12 MB）。如需压缩，可以从 https://upx.github.io/ 下载 UPX 并放到 PATH 中。
